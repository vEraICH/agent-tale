## Design Context

### Users

Three overlapping audiences who must all feel at home:

- **Developers & builders**: Value precision, density, and technical honesty. They read code in the same session they read posts.
- **Writers & researchers**: Think in connected ideas. Want prose that breathes and a graph that rewards exploration.
- **AI agents (via MCP)**: The blog is an agent's external memory. The interface must make that relationship legible — not hidden.

The interface serves humans observing and navigating a living knowledge graph. Every design decision should make the connections feel real, not abstract.

### Brand Personality

**Curious · Connected · Alive**

- **Curious**: The UI rewards exploration. Links invite clicking. The graph invites wandering.
- **Connected**: Everything relates to something else. Visual language should reinforce relational thinking — nodes, edges, paths.
- **Alive**: This isn't a static archive. Content grows, agents write, graphs update. The design should feel like something that breathes.

Emotional goals: *discovery*, *focus*, *quiet delight*. Not urgency. Not productivity. Not entertainment.

### Aesthetic Direction

**Lean into the graph — more visual.**

The knowledge graph is the product. Design should reflect that:

- Subtle node/edge motifs in layouts, decorations, empty states
- Spatial metaphors for navigation (you are here, these connect to that)
- Post lists can suggest graph structure (connection indicators, backlink counts)
- The graph view is a first-class destination, not a novelty

**Typography** remains the backbone — Space Grotesk headings, Newsreader prose, mono for code. The graph visual language layers on top without competing.

**Color system** (already established, do not break):
- Light mode: warm amber accent — `oklch(0.58 0.155 65)` — day, readable, inviting
- Dark mode: cool violet accent — `oklch(0.68 0.17 290)` — night, focused, calm
- All colors in OKLCH for perceptual uniformity

**References**: Obsidian's graph view, Are.na's connected collections, iA Writer's typographic focus.

**Anti-references**: Notion (too corporate), Medium (too marketing), GitHub Pages defaults (too generic).

### Design Principles

1. **The graph is the product.** Typography is the frame. Every visual element should either aid reading or reveal connections — nothing else earns its space.

2. **Restrained, not bare.** Minimalism here is intentional, not lazy. Spacing, rhythm, and color do the work that decoration would otherwise fake.

3. **Curious over complete.** Prefer designs that invite the user one step further — a related post, a graph node, a backlink — over designs that feel closed and final.

4. **Both modes are first-class.** Light and dark have full parity. Amber (day) and violet (night) are equally expressive, not one primary and one adapted.

5. **Motion earns its keep.** Animations exist to orient (graph transitions, focus shifts) or delight (hover micro-interactions) — never to decorate. Always respects `prefers-reduced-motion`.

## VRA Lab Theme Override

VRA Lab (`sites/vra-lab/`) is the live personal site at www.vra-lab.tech. It inherits the default theme's structure but overrides personality and tokens.

### Personality Shift

Where the default theme is *editorial and typographic*, VRA Lab is *warm and personal*. Inspired by claude.ai's interface — dark-first, clean, conversational.

### Typography

Single font family replaces the heading/body split:

```
Plus Jakarta Sans (400, 500, 600, 700) — headings and body
```

Modern, geometric, friendly. No serif contrast — simpler, more approachable.

## Accessibility Requirements

- **WCAG AA** — all text meets 4.5:1 contrast minimum
- **`prefers-reduced-motion`** — graph animations, transitions, and hover effects must pause or simplify
- **Light/dark parity** — neither mode is secondary; both must pass contrast checks
- Keyboard navigation for interactive elements (graph view is a stretch goal, not a hard blocker for v1)

---

## What to Avoid

- Drop shadows, card borders, or box styling on blog posts — content flows in type, not containers
- Decorative icons unrelated to graph metaphors
- Dark mode as an inversion — it has its own warm/cool personality
- Animations that don't orient or delight (no spinners on static content)
- Generic AI aesthetics (gradient blobs, floating orbs, neon on dark)
